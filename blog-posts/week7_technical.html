The point of an SQL Injection attack is to compromise a database, which can include user names, passwords, text, etc. A code injection attack is basically an unwanted request for a website to run malicious computer code. The objective is to fool the database system into code that will reveal sensitive information or otherwise compromise the server.

After a site is identified as being vunerable, a malicious user can work to alter SQL statements to serve a different purpose than what was originally intended. Once someone has gained unauthorized access to your site, they can damage it any number of ways, including:

  • Redirecting visitors to another site by using JavaScript
  • Loss of data confidentiality
  • Loss of data integrity
  • Loss of data
  • Compromise of the entire network
  • Disabling web pages or introducing errors
